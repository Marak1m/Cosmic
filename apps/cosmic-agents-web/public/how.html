How it works

Structure over chaos. Convergence over chatter.

Cosmic turns a single prompt into a small crew of specialists orbiting one Leader. Turns are routed, memory is bounded, and the run ends the moment the answer is good enough.

Live topology Leader + specialists in orbit
One Leader. Many specialists. One converged answer.
  1. 1

    Plan the crew

    A planner reads your prompt and hires up to five specialists plus a Leader — each with a narrow focus and the tools it needs.

  2. 2

    Emit the Task Brief

    The Leader's first turn is a structured brief D: objective, constraints, and 3–6 subtasks assigned to specialists. Pinned outside rolling memory so it's always in context.

  3. 3

    Route every turn

    A Structured Speaker Allocator (SSA) picks the next agent — either by explicit nomination in the previous message, or by a small selector call scoring candidates on dependency-readiness, peer references, and topic fit.

  4. 4

    Share bounded memory

    All agents read from one rolling FIFO window Mt. Peers can reference each other ("Building on Flight_Routing_Analyst's routing…") and flag real conflicts across subtasks.

  5. 5

    Heartbeat + terminate

    Every few turns the Leader scores coverage C(t), consistency H(t), and stability τ. When the crew converges, the Leader emits TERMINATE and a synthesizer produces the final user-facing answer.
